JS 执行上下文指当前执行环境中的变量, 函数声明, 参数(arguments), 作用域链, this 等信息. 分为**全局执行上下文**, **函数执行上下文**, 两张的区别在于全局执行上下文只有一个, 函数执行上下文在每次函数调用时候都会创建一个新的函数执行上下文.
```javascript
// 上下文示例
const ExcutionContextObj = {
  VO: window, // 变量对象
  ScopeChain: {}, // 作用域链
  this: window
}
```

执行上下文的生命周期:
-   创建阶段  
	-   生成变量对象
		-   创建arguments
		-   扫描函数声明
		-   扫描变量声明
	-   建立作用域链
	-   确定this的指向
-   执行阶段
	-   变量赋值
	-   函数的引用
	-   执行其他代码
**执行上下文栈**
JS 执行上下文遵循栈结构的执行方式, FILO 先进后出
